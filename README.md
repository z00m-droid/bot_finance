# Telegram-бот для учёта финансов

Бот записывает финансовые операции (доход/расход) и переводы между своими счетами (транзит)
в Google Sheets через Service Account. Дополнительно поддерживает автоматическую "копилку":
округление расходов до заданного шага с автопереводом разницы на накопительный счёт.
Отчётов и статистики нет (архитектура это позволяет добавить позже) — бот только пишет данные
и читает то, что нужно для интерфейса и подтверждений (справочники, баланс).

## Структура проекта

```
project/
├── main.py                # точка входа
├── config.py              # все настройки (кроме секретов)
├── handlers.py             # обработчики сообщений и callback
├── keyboards.py            # клавиатуры
├── google_sheets.py        # работа с Google Sheets (gspread)
├── states.py               # собственная FSM в памяти
├── requirements.txt
├── credentials.json        # Service Account (нужно добавить самостоятельно)
├── .env                    # секретные переменные (нужно создать из .env.example)
├── .gitignore
└── README.md
```

## Структура Google-таблицы

- Лист **Настройки**:
  - категории расходов — `B2:B29`, категории доходов — `B31:B34`;
  - настройки копилки — `H3:K1000`, столбцы: Счёт | Округлять (да/нет) | Шаг | Счёт накопления.
- Лист **Счета**: список счетов в первом столбце (`A2:A…`), пустые строки игнорируются.
  Баланс читается из столбца **E** ("Текущий баланс") — в таблице это формула
  (`=Стартовый баланс ± SUMIFS по Операции и Транзит`), поэтому бот не считает баланс сам,
  а просто перечитывает ячейку после записи (с небольшой паузой на пересчёт формулы).
- Лист **Операции**: строки добавляются в столбцы `A:F` — Дата, Тип, Категория, Счёт, Сумма, Комментарий.
- Лист **Транзит**: строки добавляются в столбцы `A:E` — Дата, Откуда, Куда, Сумма, Комментарий.
  Сюда попадают и переводы, которые пользователь создаёт вручную через кнопку **🔁 Транзит**,
  и автоматические записи копилки. На доходы/расходы эти суммы не влияют.

Все диапазоны и названия листов вынесены в `config.py`. При изменении структуры таблицы
достаточно поменять значения в `config.py` — логика кода не требует изменений.

## Автоматическая копилка (округление расходов)

Работает только для операций типа **Расход**. После записи расхода бот проверяет строку
счёта в таблице настроек копилки (`Настройки!H3:K1000`):

1. Если для счёта **Округлять = нет** (или строки для счёта нет вовсе) — копилка не срабатывает.
2. Если **да** — сумма расхода округляется вверх до ближайшего кратного **Шагу**.
   Например, расход 150 ₽ при шаге 100 округляется до 200 ₽, разница — 50 ₽.
3. Если разница равна 0 (сумма уже кратна шагу) — запись в "Транзит" не создаётся.
4. Если разница больше 0 — в лист **Транзит** добавляется строка: Откуда = счёт списания,
   Куда = **Счёт накопления** из настроек, Сумма = разница, Комментарий = "Автоокругление".
5. Расход в "Операции" остаётся равным исходной сумме — копилка не искажает статистику расходов,
   это отдельный внутренний перевод.

Настройки читаются из таблицы каждый раз заново (без кеша), так что изменения в Google Sheets
применяются к следующей же операции.

## Установка

1. Создайте проект в Google Cloud, включите Google Sheets API и Google Drive API.
2. Создайте Service Account, скачайте JSON-ключ и сохраните его как `credentials.json` в корне проекта.
3. Откройте вашу Google-таблицу на редактирование для email-адреса Service Account
   (поле `client_email` в `credentials.json`).
4. Установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

5. Скопируйте `.env.example` в `.env` и заполните:

   ```bash
   cp .env.example .env
   ```

   - `BOT_TOKEN` — токен от [@BotFather](https://t.me/BotFather);
   - `SPREADSHEET_ID` — ID таблицы из её URL;
   - `ALLOWED_USER_IDS` — Telegram User ID через запятую (узнать свой ID можно у [@userinfobot](https://t.me/userinfobot));
   - `GOOGLE_CREDENTIALS_PATH` — путь к `credentials.json` (по умолчанию — корень проекта).

6. Запустите бота:

   ```bash
   python main.py
   ```

## Сценарий работы

Главное меню (постоянная кнопка **Меню**) показывает две inline-кнопки:

### ➕ Добавить операцию
1. Дата → тип операции (Доход/Расход) → категория → счёт → сумма → комментарий.
2. Операция сразу записывается в лист **Операции**.
3. Если это расход и для счёта включена копилка — создаётся автоматическая запись в "Транзит"
   (см. раздел выше), о чём бот сообщает отдельной строкой в подтверждении.
4. Бот присылает подтверждение с текущим балансом счёта (уже с учётом копилки, если она сработала).

### 🔁 Транзит
1. Дата → счёт списания ("Откуда") → счёт зачисления ("Куда", без счёта списания в списке —
   перевести самому себе нельзя) → сумма → комментарий.
2. Перевод записывается в лист **Транзит**.
3. Бот присылает подтверждение с балансами обоих счетов.

На каждом шаге обоих сценариев доступна кнопка **❌ Отмена**, полностью очищающая временные
данные пользователя. После завершения операции, перевода или отмены бот автоматически
присылает главное меню — заново нажимать "Меню" не нужно.

## Постоянный запуск (деплой)

Бот использует long polling, поэтому ему не нужен открытый порт или домен — только
постоянно работающий процесс с доступом в интернет. Ниже два готовых способа.

### Вариант A: VPS + systemd (рекомендуется)

1. Скопируйте проект на сервер, например в `/opt/telegram_finance_bot`.
2. Создайте пользователя и виртуальное окружение:

   ```bash
   sudo useradd -r -s /bin/false botuser
   cd /opt/telegram_finance_bot
   python3 -m venv venv
   ./venv/bin/pip install -r requirements.txt
   sudo chown -R botuser:botuser /opt/telegram_finance_bot
   ```

3. Убедитесь, что `.env` и `credentials.json` лежат в `/opt/telegram_finance_bot`.
4. Скопируйте юнит и запустите сервис:

   ```bash
   sudo cp deploy/telegram-finance-bot.service /etc/systemd/system/
   sudo systemctl daemon-reload
   sudo systemctl enable --now telegram-finance-bot
   ```

5. Проверка и логи:

   ```bash
   sudo systemctl status telegram-finance-bot
   tail -f /var/log/telegram_finance_bot.log
   ```

Сервис автоматически перезапустится при падении процесса и при перезагрузке сервера.

### Вариант B: Docker

Подходит для любого сервера с установленным Docker, не требует настройки Python-окружения.

1. Убедитесь, что `.env` и `credentials.json` лежат в корне проекта.
2. Соберите и запустите:

   ```bash
   docker compose up -d --build
   ```

3. Логи и статус:

   ```bash
   docker compose logs -f
   docker compose ps
   ```

`restart: unless-stopped` в `docker-compose.yml` гарантирует автозапуск контейнера
после перезагрузки сервера (при условии, что Docker сам настроен на автозапуск —
это по умолчанию для большинства дистрибутивов).

### Вариант C: Railway (или другой PaaS без загрузки файлов)

Railway разворачивает проект прямо из GitHub-репозитория и держит процесс запущенным
постоянно — отдельно настраивать systemd или Docker не нужно, платформа делает это сама.
Проблема только в `credentials.json`: это файл, а Railway хранит только переменные окружения.

Решение — передать содержимое файла через переменную `GOOGLE_CREDENTIALS_JSON`, которую
уже поддерживает `google_sheets.py`.

1. Откройте `credentials.json` и скопируйте его содержимое целиком, в одну строку.
   Проще всего сделать это командой (локально, там где лежит файл):

   ```bash
   python3 -c "import json; print(json.dumps(json.load(open('credentials.json'))))"
   ```

   Это выведет весь JSON в одну строку — именно её нужно скопировать.

2. В Railway откройте ваш сервис → **Variables** → **New Variable**:
   - имя: `GOOGLE_CREDENTIALS_JSON`
   - значение: вставьте скопированную строку целиком.

3. Остальные переменные (`BOT_TOKEN`, `SPREADSHEET_ID`, `ALLOWED_USER_IDS`) добавьте туда же,
   как вы уже сделали. `GOOGLE_CREDENTIALS_PATH` можно не добавлять — он не используется,
   если задан `GOOGLE_CREDENTIALS_JSON`.

4. Убедитесь, что в `.gitignore` остаются `.env` и `credentials.json` — они не должны
   попадать в репозиторий ни при каких обстоятельствах, все секреты только через
   переменные окружения Railway.

5. Сохраните переменные — Railway автоматически передеплоит сервис. В логах должно
   пропасть сообщение об отсутствии `credentials.json`.

## Расширение проекта

Архитектура допускает добавление без переработки существующего кода:

- редактирование и удаление операций/переводов — новые обработчики в `handlers.py` и новые состояния в `states.py`;
- отчёты и статистику — новый модуль чтения (`google_sheets.py` уже разделяет чтение и запись);
- дополнительные меню и команды — новые inline-клавиатуры в `keyboards.py`;
- режим фиксированного пополнения копилки (вместо/вместе с округлением) — добавляется как ещё
  один столбец в таблицу настроек копилки и ветка в `apply_autoround_if_needed` (`handlers.py`);
- несколько таблиц или пользователей — расширение `config.py` (например, словарь `user_id → spreadsheet_id`)
  без изменения логики `handlers.py`.
